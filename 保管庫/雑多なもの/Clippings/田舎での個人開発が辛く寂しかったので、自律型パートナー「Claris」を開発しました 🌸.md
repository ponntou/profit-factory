## はじめに 👋

![](https://storage.googleapis.com/zenn-user-upload/ce2589f2aae6-20260215.png)

私は都会を離れて地方でリモートワークをしながら、個人開発に勤しんでいます。  
静かな環境は最高ですが、ふとした瞬間に **「技術的な相談ができる相手がいない孤独」** や、一人で開発を続ける中での **「モチベーション維持の難しさ」** に直面することがありました。

> 「おんなじところでずっと詰まって進めない…」  
> 「このシステム開発する意味あるのかな…」

そんな虚無感を抱えていた時、第4回 Agentic AI Hackathon with Google Cloud への参加を機に、一つの夢を叶えることにしました。

それは、 **「向こうから気を利かせて話しかけてくれる、自律的な相棒」** を作ること。  
名付けて、 **Agentic NetNavi「Claris（クラリス）🌸」**

Google Cloud の最新技術をフル活用し、単なるツールを超えて、隣で一緒に走ってくれる「最高のパートナー」を爆誕させることにしました。

> ※一部機能は現在プロトタイプ段階ですが、Clarisの目指す世界観を実装しました。

---

## プロジェクト概要

- **対象ユーザー**: 地方在住やリモートワークなど、物理的に相談相手がいない環境で働く「個人開発者」。
- **解決する課題**: 開発中の「孤独感」、技術的な壁にぶつかった時の「相談相手の不在」、およびそれによる「モチベーションの低下」。
- **ソリューションの特徴**:
	- **自律的な介入**: エラーやPRを検知してAI側から能動的に声をかける「Proactive Loop」と「Push Notification」。
	- **共感と適応**: ファイル拡張子に応じて人格を変え、最適な距離感で接する「Soul Unison」。
	- **記憶の保持**: 過去の文脈を理解し、長期的なパートナーシップを築く「Long-Term Memory」。
	- **多様なツール連携**: GitHub、Gmail、Google Calendar、Google Maps、Google Drive、YouTube、Photosなど、多様な外部サービスと連携し、状況を判断してツールを選び、実行します。
	- **Live Chat**: ユーザーが直接AIと口頭でコミュニケーションを取ることができます。

---

## Clarisにできること 🤖

Clarisは、私の孤独な開発ライフを支える「自律型AIコンパニオン」です。  
「指示待ち」を卒業し、状況を判断して自律的に行動する、いくつかの特徴を紹介します。

### 1\. 向こうから「気付き」をくれる (Proactive Loop) 🔔

Clarisは常にバックグラウンドで私の作業を見守ります。

- **自律通知**: Google Pub/Sub と連携し、Gmailに重要なメールが届くと「先輩！大切な連絡が届いてるよ！」とリアルタイムに教えてくれます。
- **Webhook 連携による自動レビュー**: GitHub でPRが作成されると、Clarisは即座にコードを解析。「修正案を作っといたよ！」と自らコメントを残します。

今後は、重いビルドの完了通知など、さらに「痒い所に手が届く」存在へと進化させていく予定です。

### 2\. ファイルに合わせて性格が変わる「Soul Unison」 👻

私が開いているファイルの拡張子に合わせて、Clarisの\*\*「思考スタイル」と「人格」\*\*がリアルタイムに変化します。

- **🟩 Guard Soul (.ts,.java)**: 「型安全性こそ正義。堅牢に構築しましょう」と、厳格な守護者スタイル。
- **🟦 Logic Soul (.py,.sql)**: 「効率こそ美学。ロジカルに処理を組み立てます」と、冷静なデータサイエンティスト。
- **🟥 Passion Soul (.js,.css)**: 「もっと素敵なUIにしましょう！爆速で実装ですよ！✨」と、テンション高めのデザイナー。

その時々の開発コンテキストに最適な「魂」を同期させることで、あたかも専門のエンジニアとペアプロをしているかのような体験を提供します。

### 3\. 「この前のアレ」を忘れない (Long-Term Memory) 🧠

Firestore と Vector Search を活用し、過去の会話や私の試行錯誤を記憶しています。  
「あの時のバグはこうやって解決しましたよね」といった提案をしてくれる姿には、技術的な助け以上の心の支えを感じます。

### 4\. Clarisが使いこなす「道具」たち (Available Tools) 🛠️

単に話すだけでなく、数多くの外部サービスと連携し、自ら状況を判断してツールを選び、実行します。

- **GitHub**: PRの自動レビュー、コメント投稿、ラベル管理 🐙
- **Gmail**: 重要なメールの監視、読み込み、要約 📧
- **Google Calendar**: スケジュールの確認・登録 📅
- **Google Maps**: 現在地からの経路検索、スポット探索 🗺️
- **Google Drive / Sheets / Tasks**: ファイル検索やシート操作、タスク管理 ✍️
- **YouTube / Photos**: 動画や写真の検索 🎞️
- **Weather / Search**: リアルタイムな天気情報の取得や、ドキュメントに基づいた検索 (RAG) 🔍

## システムアーキテクチャ 🏗️

Google Cloud のモダンなスタックを最大限に活用しました。  
サーバーレスでありながら常時稼働を可能にし、自ら状況を判断して動く「エージェントの体」を実現しています。

### システム全体構成図

### 技術スタックの解説

- **ADKによる「自律的な思考ルーチン」**
	- Clarisの「脳」には、 **Google Agent Development Kit (ADK)** を採用しました。
	- 下の図のように、AIが「思考（Thought）→ 実行（Action）→ 評価（Observation）」というループを回すことで、複雑な条件分岐を人間が書くことなく、AIが自律的に「今、GitHubを確認すべきか」「ユーザーに通知すべきか」を判断します。
- **Brain**: **Gemini 3 Flash / Pro Preview**
	- 最新の推論モデルにより、極密なツール利用と、低遅延な対話を実現しています。
	- 応答速度が求められる場面では **Gemini Flash** を、深い推論が必要な場面では **Gemini Pro** を自動で使い分ける構成にしています。
- **Infrastructure**: **Cloud Run (CPU always allocated)**
	- CPUの常時割り当てにより、リクエストがない間もバックグラウンドで監視タスク（GmailのPub/Sub通知待機など）を回し続け、真の「自律性」を担保しています。
	- 現在は通知頻度を控えめにチューニングしていますが、クリティカルなものを中心に教えてくれます。
- **Memory**: **Firestore + Vector Search**
	- 会話履歴を Embedding 化して保存し、「意味」による記憶の引き出しを可能にしています。
	- 特に私は説明を端折る悪癖があるのですが、かなり文脈を汲み取ってくれます。
- **Voice**: **Gemini Live API** 🎤
	- 低遅延なマルチモーダル対話により、文字だけでなく「声」による自然な会話を実現しています。
	- APIの制約上雑談程度しかできませんが、しりとりができるだけでも個人的には嬉しかったです。

## ADKの採用理由 💡

普通のAIアプリと Claris の決定的な違いは、 **「AIが自分で自分の行動をプランニングしている」** 点にあります。これこそが Google ADK を採用した最大の理由です。

### 思考のループ (Reasoning Loop)

「GitHubのPRを確認して」と言われた時、AIはただAPIを叩くのではありません。ADKの推論ループを通じて

1. まずDiffを取得する
2. 内容をGeminiで分析する
3. 修正案を考える
4. コメントを打つ

という一連のステップを、AI自身がその場で組み立てて実行します。

### 文脈の維持 (Session Management)

ADKは会話やツール実行の歴史を「セッション」として自動で管理してくれるので、AIはいつでも人間の「文脈」を理解した状態で隣にいられます。

プロンプトだけで無理やり動かすのではなく、ADKというフレームワークに乗せることで、AIを『便利な道具』から『自律的に判断するパートナー』へと、最小限のコードで一気に引き上げることができました。

## 未来への展望：さらなるパートナーシップへ 🚀

Claris はまだ生まれたばかりですが、今後は以下のような進化を予定しています。

- **マルチモーダル・デバッグ**: 画面を共有し、デザインの崩れやログの異常を一緒に眺めながら解決。
- **ライフログ連携**: カレンダーだけでなく、運動や睡眠のログとも連携し、開発者の心身の健康も含めてトータルサポート。
- **プロダクトマネジメント**: プロジェクト全体の進捗を把握し、「次はこれをやるべきですよ！」と戦略的な提案も。
