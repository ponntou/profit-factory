## はじめに

コーディングエージェントのデモは魔法のように見えます。しかし、実際のコードベースに向けると：

- コンテキストウィンドウがすぐに埋まる
- 古いコードに対してハルシネーションが起きる
- 処理が遅すぎて、grepした方が早い

大規模なRust/Python/TSリポジトリでAIワークフローを構築する中でこの壁にぶつかったので、自分のスタックに本当に欲しかったものを作りました： **ASTベースの超軽量な組み込みMCP** です。

[`cocoindex-code`](https://github.com/cocoindex-io/cocoindex-code) というツールで、トークン消費を約70%削減し、待ち時間も大幅に短縮できます。

Claude、Codex、Cursor、その他MCP対応のコーディングエージェントを使っている方は、ぜひ読んでみてください。

## コアアイデア：AST + インクリメンタルインデックス

よくある「コードRAG」構成は、ベクトルDBの構築、ETLの作成、スキーマドリフトへの対応、チャンキングのチューニングなど、インフラプロジェクトのようになりがちです。

`cocoindex-code` は真逆のアプローチを取ります：

- **組み込みMCP** ：ローカルでMCPサーバーとして動作し、別途DBの運用が不要です。
- **ASTベースのインデックス** ：Tree-sitterでコード構造を解析し、関数・クラス・ブロック単位で意味のあるチャンクを取得します。
- **インクリメンタル更新** ：RustベースのCocoIndexエンジン上に構築されており、変更されたファイルのみ再インデックスします。
- **実用的なマルチ言語対応** ：Python、JS/TS、Rust、Go、Java、C/C++、C#、SQL、Shellなど。

エージェントに質問すると、必要なコードだけを正確に取得し、 **コンテキストウィンドウを爆発させません** 。

## すぐに使える機能

MCPを追加するだけで以下が利用可能です：

- **セマンティックコード検索ツール** ： `search(query, limit, offset, refresh_index)` をMCPツールとして提供。
- **トークンの即時削減** ：関連するコードチャンクのみがプロンプトに入るため、ファイル全体を渡す必要がない。
- **高速** ：インクリメンタルインデックス + Rustエンジンで、更新はほぼ瞬時。
- **ローカル埋め込みがデフォルト** ： `sentence-transformers/all-MiniLM-L6-v2` をローカルで使用。APIキー不要。
- **オプションで強化可能** ：LiteLLM対応の任意の埋め込みモデル（OpenAI、Gemini、Mistral、Voyage、Ollamaなど）に切り替え可能。

つまり、「普通のコーディングエージェント」から「コードベースを理解したエージェント」へ約1分でアップグレードできます。

## 1分セットアップ（Claude / Codex / OpenCode）

まず `uv` をインストールします：

```
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Claudeの場合

```
claude mcp add cocoindex-code \
  -- uvx --prerelease=explicit --with \
  "cocoindex>=1.0.0a16" \
  cocoindex-code@latest
```

### Codexの場合

```
codex mcp add cocoindex-code \
  -- uvx --prerelease=explicit --with \
  "cocoindex>=1.0.0a16" \
  cocoindex-code@latest
```

### OpenCodeの場合

```
opencode mcp add
# MCP server name: cocoindex-code
# type: local
# command:
# uvx --prerelease=explicit --with cocoindex>=1.0.0a16 cocoindex-code@latest
```

これだけです。リポジトリをエージェントに向ければ、コードベース全体のセマンティック検索がMCPツールとして使えるようになります。

## search MCPツールの仕組み

接続すると、MCPは `search` ツールを公開します：

```
search(
  query: str,        # 自然言語またはコードスニペット
  limit: int = 10,   # 1-100
  offset: int = 0,   # ページネーション
  refresh_index: bool = True  # クエリ前に再インデックス
)
```

各結果には以下が含まれます：

- ファイルパス
- 言語
- コード内容
- 開始/終了行番号
- 類似度スコア

特に役立つユースケース：

1. 「Xの実装はどこ？」— 似た名前の関数が5つあるとき。
2. 「JWTリフレッシュに関わる認証ロジックを全部見せて」
3. 「このスタックトレースにマッチするコードを見つけて」

インデックスはインクリメンタルに更新されるので、リファクタリング後も即座に新しいコードに対してエージェントを使えます。

## 対応言語

C, C++, C#, CSS/SCSS, Go, HTML, Java, JavaScript/TypeScript/TSX, JSON/YAML/TOML, Kotlin, Markdown/MDX, Pascal, PHP, Python, R, Ruby, Rust, Scala, Solidity, SQL, Swift, XML

`__pycache__` 、 `node_modules` 、 `target` 、 `dist` などのノイズの多いディレクトリは自動で除外されます。ルートパスは`.cocoindex_code/` 、`.git/` 、またはカレントディレクトリから自動検出されるので、通常は環境変数の設定も不要です。

## 埋め込みモデル：無料で始めて、後からスケール

デフォルトではローカルのSentenceTransformersモデルを使用：

- `sbert/sentence-transformers/all-MiniLM-L6-v2`
- APIキー不要、課金なし、完全ローカル

より強力なセマンティック理解が必要な場合は、 `COCOINDEX_CODE_EMBEDDING_MODEL` でLiteLLM対応の任意のモデルに切り替えられます：Ollama、OpenAI、Gemini、Mistral、Voyage、Cohere、AWS Bedrock、Nebiusなど。

**まずは無料のローカルで始めて、必要になったらアップグレード。**

## 大規模/エンタープライズコードベースへの対応

内部的には [CocoIndex](https://github.com/cocoindex-io/cocoindex) というRustベースのインデックスエンジンを使用しており、大規模なインクリメンタルデータワークフロー向けに設計されています。

組織での利用では：

- チーム間でインデックスを共有し、各マシンでの再インデックスを回避
- ブランチ重複排除で無駄な作業を削減
- CocoIndex上のより大規模なデータ/インデックスプラットフォームの一部として運用可能
