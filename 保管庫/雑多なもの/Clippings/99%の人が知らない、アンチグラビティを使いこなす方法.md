### Q: AntiGravityを99%の人より効果的に使うための方法は？

*AntiGravityの隠れた機能と「Flow」という4ステップのフレームワーク（Frame, Layout, Orchestration, World）を活用し、 ==AIエージェントを並行稼働させ、MCPサーバーで外部サービスと連携==* することで、高品質なアプリを迅速に開発・公開できます。

### Q: 「Flow」フレームワークの各ステップの概要は？

- **Frame**: 解決する問題を具体化し、Claudeと対話して要件を洗練する。
- **Layout**: デザインのインスピレーションを取り入れ、ブランドガイドラインや画像をアップロードしてデザインを定義する。
- **Orchestration**: 複数のAIエージェントを並行稼働させ、成果物をレビュー・修正しながらアプリを構築する。
- **World**: 構築したアプリをGitHubとVercelにデプロイし、世界に公開する。

Googleの革新的なAIコーディングツール「AntiGravity」を、単なるチャットボットとしてではなく、 **プロレベルのアプリケーションを10倍速く構築** するためのシステムとして活用する方法を解説します。本コンテンツでは、 **Flowフレームワーク（Frame, Layout, Orchestration, World）** という具体的な4ステップを通じて、アイデアの明確化からデザインの最適化、そしてGitHubとVercelを利用した **アプリの公開とデプロイ** までを実践的に学びます。特に、複数のAIエージェントを並行して実行し、デザインのブランドガイドラインやMCPサーバー（Model Context Protocol）を活用して、 **競合他社よりも高品質なアプリ** を効率的に開発する **隠された機能と高度なテクニック** を習得できます。コーディング経験がない方でも、この体系的なアプローチを通じて、AIを活用したソフトウェア開発の最前線に立つための具体的な道筋が得られます。

## 1\. AntiGravityの概要とFlowフレームワークの紹介 \[0\]

![thumbnail](https://resource.lilys.ai/thumbnails/QZCYFJTv8jI/0.jpg)
1. **AntiGravityの定義と利点**:
	1. Googleの革新的なAIコーディングツールであり、正しく使用すれば *ゲームチェンジャー* となる \[0\]
	2. 多くの人々はチャットボットのように使用しているが、筆者は **プロレベルのアプリを10倍速く構築** するシステムを発見した \[1\]
	3. AntiGravityは、Microsoft Wordがドキュメント、Excelがスプレッドシートであるように、 **ソフトウェア構築のためのツール** である \[9\]
	4. 内部には、 **Gemini 3** （最も強力なAIデザインツール）、 **Claude 4.5 Sonnet** 、Codespaces、および **ChatGPTモデル** へのアクセスが含まれている \[12\]
	5. AIエージェントのためのネイティブシステムへのアクセスも提供され、他者よりも *圧倒的な優位性* をもたらす \[15\]
2. **学習アプローチとFlowフレームワーク**:
	1. 99%の人々より優位に立つ最良の方法は、アプリを構築しながら知識を習得することである \[18\]
	2. アプリ構築には、 **Flow** と呼ばれる4ステップのフレームワークを使用する \[21\]
	3. F: Frame (フレーム)
	4. L: Layout (レイアウト)
	5. O: Orchestration (オーケストレーション)
	6. W: World (ワールド)

## 2\. Flowフレームワークの最初のステップ: Frame (フレーム) \[22\]

![thumbnail](https://resource.lilys.ai/thumbnails/QZCYFJTv8jI/89.jpg)
1. **AntiGravityのダウンロードとワークスペース設定**:
	1. Flowを開始する前に、AntiGravityをダウンロードする必要がある（Mac OS用） \[22\]
	2. AntiGravityのインターフェースは、右側にエージェントとのチャット、中央にコード表示、左側にファイルが表示される \[24\]
	3. ワークスペースはプロジェクト単位で機能し、「フォルダを開く」をクリックして新しいフォルダ（例: Teddy）を作成し、それをワークスペースとして使用する \[29\]
2. **Frameの目的と問題の明確化**:
	1. Frameの「F」は、 **解決しようとしている問題について具体的にする** ことを意味する \[42\]
	2. アインシュタインの言葉のように、問題解決に55分を費やすべきである \[44\]
	3. **実践例**: 家計の請求書ダッシュボードアプリを構築するアイデアをClaudeに提示し、思考に挑戦してもらい、問題を洗練させる \[45\]
	4. Claudeはこのような *対話形式のやり取り* に優れているため、このステップに適している \[52\]
	5. **プロンプトの重要性**: 初期プロンプトが具体的であればあるほど、アプリの品質は向上する（「Joeアプリ」と「プロアプリ」の違いとなる） \[54\]
3. **AI開発のためのSOP作成**:
	1. Claudeが提示した質問に回答し、問題が洗練された後、 **AIが開発するためのSOP** （最大500文字）をClaudeに設計させる \[57\]
	2. Claudeがこのプロンプトを生成した後、AntiGravityに戻る \[60\]
	3. AntiGravityの右側チャットウィンドウで、 **Planningエージェント** を選択し、思考させる（初期段階で推奨される） \[61\]

## 3\. Flowフレームワークの第二・第三ステップ: LayoutとOrchestration \[64\]

![thumbnail](https://resource.lilys.ai/thumbnails/QZCYFJTv8jI/225.jpg)

### 3.1. Layout (レイアウト) - デザインのインスピレーションとブランドガイドラインの適用 \[64\]

1. **デザインのインスピレーションの取得**:
	1. Flowフレームワークの2番目の「L」はLayoutであり、デザインのインスピレーションを得る \[64\]
	2. Dribbbleなどのサイトでデザインのインスピレーション（例: 請求書ダッシュボード）を見つける \[68\]
	3. AntiGravityは **画像を受け取ることができる** ため、インスピレーション画像をコピーして貼り付ける \[70\]
2. **ブランドガイドラインのアップロードと適用**:
	1. Layoutセクションの左側にあるデザインセクションで、 **ブランドガイドラインやクールな画像** をアップロードできる \[73\]
	2. 既存のブランドガイドライン（例: PDF）をファイルにドロップインできる \[75\]
	3. **カスタムデザイン指示の作成**:
	4. 左側で新しいファイル（例: `gemini.md` ）を作成し、 **デザインに関する指示リスト** を記述できる \[77\]
	5. フォントやタイポグラフィなど、望むものを指定できる \[79\]
	6. ブランドのロゴやクライアントのロゴなどの画像も、左側にドラッグアンドドロップでアップロードできる \[80\]
	7. **カスタムブランドガイドラインの作成**:
	8. PDFが大きすぎる場合、左側で新しいファイル（例: `brand_guidelines` ）を作成し、ガイドライン情報を直接入力できる \[88\]
	9. Layoutセクションは、すべてのレイアウト関連事項を左側で **事前に定義** できる場所として機能する \[85\]

### 3.2. Orchestration (オーケストレーション) - 構築と高度なエージェント管理 \[99\]

1. **成果物(Artifacts)の活用と注釈(Annotations)**:
	1. Flowフレームワークの3番目の「O」はOrchestrationであり、すべてを構築する段階である \[99\]
	2. 多くの人が見逃す機能として、 **成果物(Artifacts)** がある \[100\]
	3. AntiGravityは、実装計画、目標説明、ユーザーレビュー、提案された変更などを成果物として提供する \[101\]
	4. **注釈機能**: コードやタスクリストの特定の行にコメントを追加できる \[102\]
	5. 例: スタイリングに関する行に「提供されたブランドガイドラインを使用するように」というコメントを追加する \[103\]
	6. 例: アニメーションに関するタスクに「アニメーションを特に豪華でインタラクティブにするように」というコメントを追加する \[111\]
	7. この *往復のやり取り* により、AntiGravityからの出力品質が大幅に向上する \[114\]
2. **並列エージェント実行とAgent Manager**:
	1. AntiGravityが他と一線を画す機能の一つは、 **Agent Manager** である \[116\]
	2. Agent Managerでは、複数のプロジェクト（例: Thumbs, Anti-gravity）を左側で確認できる \[124\]
	3. **並列エージェントの作成**: ワークスペースにカーソルを合わせ「+」をクリックすることで、プロジェクト内で **複数のエージェント** を同時に作成し、異なるタスクを実行させることができる \[128\]
	4. 例: 1つのエージェントに「請求書管理のベストプラクティスに関する詳細な調査」を依頼する \[131\]
	5. 例: 別のエージェントに「最も成功している請求書管理ソフトウェアプラットフォームと共通の特徴（結果の大部分を得る20%の特徴）」を調査させる \[135\]
	6. これらのエージェントは **並行して実行** され、調査結果をプロジェクトファイル内にドキュメント（例: `research.md` 、 `competitors.md` ）として作成できる \[137\]
	7. ユーザーは、すべてのAIエージェントを含む **エージェントダッシュボードビュー** と、基本的なエージェントビュー（エディタ）を切り替えることができる \[145\]
3. **ユニバーサルインボックスと自己修正機能**:
	1. Agent Managerには **インボックス** があり、実行中のエージェントが助けを必要とするときに通知が届く \[157\]
	2. AntiGravityの最もクールな機能の一つは、 **動的にウェブブラウザでテストできる** ことである \[160\]
	3. アプリ構築中、モデルはGoogle Chromeウィンドウを開き、フォームをクリックしたりUIを試したりして、 **自己修正（self-fixing, self-healing）** を自律的に行う \[159\]
	4. エージェントがタスクを完了すると、インボックスでそれを開き、概要を確認できる \[165\]
4. **継続的なレビューと改善**:
	1. エディタに戻ると、変更の受け入れを求められる（承認する） \[171\]
	2. タスクリストの実行中に、ユーザーは再び注釈機能を使用してレビューし、改善を要求できる \[178\]
	3. 例: 月次パフォーマンスを示すグラフや、サブスクリプションのロゴ付きドロップダウンビューの追加を要求する \[180\]
	4. この共同作業的な方法で、アプリの品質を向上させることができる \[186\]

## 4\. 高度な機能: MCPサーバー、カスタマイズ、およびモデルの選択 \[195\]

### 4.1. MCPサーバー (Model Context Protocol) の活用 \[212\]

1. **MCPサーバーの定義と目的**:
	1. MCPサーバーは、 **Model Context Protocol** の略であり、アプリとAIが *楽に通信できる* ようにするための統一言語である \[217\]
	2. AntiGravityをAirtable、Vercel、GitHubなど、さまざまなプラットフォームやアプリに接続するために使用される \[218\]
	3. MCPサーバーをクリックすると、AntiGravity内で利用可能なMCPサーバーの完全なリストが表示され、一度接続すればすべてのプロジェクトで使用できる \[220\]
	4. MCPストアには、PayPal、Pine Coneなど、多くのサーバーが存在する \[223\]
2. **GitHub MCPの接続とトークン生成**:
	1. **GitHub** を追加する理由は、後でプロジェクトをデプロイできるようにするためである \[233\]
	2. GitHubアカウントを作成し、設定 > 開発者設定 > 個人アクセストークン > ファイングレイントークンに進む \[234\]
	3. 新しいトークンを生成し、名前を付け、必要な **権限** を追加する（筆者はすべて追加した後、Gemini内で制限することを好む） \[242\]
	4. 生成されたコードをAntiGravityに戻って入力し、保存すると、アプリが左側に表示される \[249\]
3. **MCPツールの制限とカスタムMCPの追加**:
	1. AntiGravityは、コンテキストウィンドウの必要性から、 **50個以下のツール** を有効にすることを推奨している \[252\]
	2. ユーザーは、SuperbaseやContext 7などのアプリのアクセスを制限し、ツール数を可能な限り低く保つべきである \[255\]
	3. **カスタムMCPの追加**: MCPストアに存在しないMCPを追加したい場合は、「View raw config」をクリックする \[258\]
	4. Context 7のように存在しないMCPの場合、その構成をコピーし、Claudeなどの言語モデルにMCPを追加するように依頼し、その結果を `MCP_config` に貼り付ける \[260\]
	5. 筆者は、NA10 MCPを例として、アクセスラインをコピーし、 `view rule config` の下部に追加し、ファイルを保存するプロセスを示している \[265\]
	6. 正常に動作すると、NA10 MCPが自動的に表示され、プロジェクトやアプリがNA10をトリガーできるようになる \[277\]

### 4.2. カスタマイズとデバッグワークフローの追加 \[281\]

1. **カスタマイズ機能の概要**:
	1. カスタマイズ機能は、ルールとワークフローを含む **プロジェクトノート** のようなものである \[281\]
	2. これらはグローバルまたはワークスペース固有に設定できる \[283\]
2. **デバッグワークフローの追加**:
	1. 筆者は、デバッグのためのワークフロー（例: Claudeリポジトリからのもの）を追加している \[288\]
	2. これにより、システムレベルの指示をいつでも追加できる \[289\]
	3. **ワークフローの作成**: ワークスペースをクリックし、名前（例: `lemon_debugging` ）を付けて、指示（例: 「これを使用してレモンを見つける」）を入力し、ファイルを保存する \[293\]
	4. **ワークフローの呼び出し**: チャットウィンドウで、 `lemon_debugging` などの名前を入力し、「問題を解決する」と指示することで、システム全体でこれらの指示を普遍的に呼び出すことができる \[304\]
	5. このシステムは、デバッグ時の最も一般的で大きな問題を特定しているため、非常に効果的である \[309\]

### 4.3. 最終的な変更とモデルの選択 \[312\]

1. **最終的なアプリの変更とGitHubへの公開**:
	1. 筆者は、アプリの背景をダークモードにし、チャットボックスを追加し、 **GitHubリポジトリを作成して公開** するようにAntiGravityに指示する \[317\]
	2. このやり取りにより、変更が **GitHubに動的に公開** され、AntiGravityでの変更がGitHubに反映される \[322\]
2. **その他の隠された機能**:
	1. **ソース管理**: 左側のタブには、コード検索、ソース管理（Source Control）がある \[324\]
	2. **拡張機能 (Extensions)**: 左側の拡張機能タブは、AntiGravityのChromeストアやAppleマーケットプレイスのようなものであり、さまざまなプラグイン（例: セマンティック検索、Ruby LSP）を検索して追加できる \[328\]
3. **ローカルホストでの確認とモデルの選択**:
	1. アプリがローカルホスト（ラップトップのCPUで実行されている状態）で実行されていることを確認する \[337\]
	2. 構築されたアプリは、 **ブランドガイドラインに完全に準拠** している \[344\]
	3. アプリはGitHubにライブで公開され、準備完了の状態である \[348\]
	4. **モデルの変更**: 右下で、使用するモデルを選択できる \[349\]
	5. **Gemini 3 Pro**: デザインに優れており、AntiGravityの組み込み機能を使用する際は、モデルがソフトウェアに最適化されているためGeminiを使用すべきである \[352\]
	6. **Claude Sonnet 4.5**: コピーライティングや自然言語に優れている（例: 記事やウェブサイトのコピー） \[357\]
	7. **GPT**: AIが間違いを犯す可能性が低い基本的なタスクや、トークンを節約したい場合に使用する \[360\]

## 5\. Flowフレームワークの最終ステップ: World (ワールド) - 公開とデプロイ \[366\]

![thumbnail](https://resource.lilys.ai/thumbnails/QZCYFJTv8jI/1349.jpg)
1. **Worldの目的**:
	1. Flowフレームワークの最終ステップ「W」はWorldであり、構築したアプリを **世界が利用できるように公開する** ことである \[366\]
2. **Vercelを使用したホスティング**:
	1. アプリはGitHub上にあるが、ホストする必要がある \[367\]
	2. Vercelなどのウェブサイトを使用してホスティングを行う \[369\]
	3. VercelをGitHubに接続し、リポジトリ（例: `glider finance dashboard` ）をインポートする \[373\]
	4. フレームワークプリセットが自動的に入力された後、「Deploy」をクリックすると、ドメインに公開される \[376\]
3. **三点測量(Triangulation)による自動更新**:
	1. このシステムは、 **Vercel、GitHub、AntiGravity** の三点測量によって機能する \[380\]
	2. AntiGravityがGitHubを更新し、GitHubがVercelを更新するため、 **瞬時に** 変更が反映される \[381\]
	3. これにより、完全に機能するアプリが完成し、共有可能となる \[384\]
