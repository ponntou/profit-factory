## はじめに

最近のAIの成長速度が著しく、それに関する記事も増えてきました

- AIの記事多すぎ
- 全部文章長すぎ
- 英語の記事も多すぎ

でした。そこで、

→そうだ、全部AIで要約したものを用意しよう

## 技術選定

### Github Actions(日々の自動化)

自動化するならPCを常に立ち上げておかなくてもよいgithubを使うのがいちばん簡単そう。あと実はちゃんと使ったことがなくて学びたかった。軽い処理なので毎日実行しても無料範囲内で動かせそうなどが理由です。

### Deepseek(Feedの要約)

調べたときは2＄まではAPIの実行が無料と出てきたのですが、実際はもう無料では使えない？ですがまぁすごい低コストなのでまぁいいかなと。  
最初はgeminiで作ったのですがすぐに無料枠の１日の上限に達したたため途中からこちらに変えました。

### Gemini(要約の要約)

リポジトリのファイルの中身を見るといったセキュリティに関わりそうなものはGeminiで動かすようにしました。一応Feedの要約程度なので漏洩しても問題ないのですが。

### Go(処理全般)

処理が早いためgithubの実行時間を減らすために採用。あともともと私がGoに慣れているというのもあります。

### Obsidian(ファイル閲覧)

mdファイルビューアーで自分好みに設定ができる最近どんどん人気が増えているアプリ。AIとmdファイルの相性が非常によく、もともとノートのテーブル表示をする方法があったが最近公式でもテーブル表示する方法が追加されたため相性がとておも良いと思いました。

## コード

コードを載せてもいいんですが、全部claude codeがやってくれたので正直誰がお願いしても同じようなものができると思います  
ざっくり指示を出した内容を書き出すとこんな感じです

- Github Actionを使って3時間おきに以下の機能を実施して
	- urls.mdに記載されたRSSフィードURL一覧から各フィードを取得し、エントリのタイトルやURLなどを抽出してObsidianの各フロントマターにそれぞれあてはめるように記入してmdファイルを作成して
	- 作成したURLの内容を確認して、Deepseekで要約をし、要約の内容を専用のフロントマターに追加して
	- 作成したファイルをObsidianを管理しているリポジトリにpushして
- Github Actionを使って1週間おきに以下の機能を実施して
	- Obsidianを管理しているリポジトリから情報を引っ張って、geminiを使って今週のAIに関する情報を要約して新しいmdファイルを作成して  
		あとはAPIキーやObsidianを管理しているリポジトリのURLやアクセストークンをGithubのEnvironment secretsに入れてそこから読み取るようにしたり、同じ記事をなんどもフィードから読み込まないようにしたりといった制御を雑にいれてもらっている感じです

## Obisidian側

Obisidianの純正テーブル機能であるBASEを使って、記事一覧を出してもらっているだけです。  
フロントマターに読んだもの(read)やピン留め(pinned)の真偽フィールドを持って、BASEの表示条件で、未読のもの(readにチェックが入っていないもの)の一覧表示や、後でまた読むもの(pinnedにチェックが入っているもの)などを表示する設定にしています

## 問題点

作ってる途中で知ったが、RSSってスクレイピングを禁止しているものがあるようです。OpenAIのRSSなどはGithub Actionsで取得する処理にたいして403が発生していました。  
運用する場合はちゃんと確認する必要がありますね。
