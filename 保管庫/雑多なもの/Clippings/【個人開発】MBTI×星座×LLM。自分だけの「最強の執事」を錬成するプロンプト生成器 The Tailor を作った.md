## はじめに：AIに「人格」はいらない、「機能」としての個性が欲しい

ChatGPTやClaudeの「System Instructions（カスタム指示）」、皆さん活用していますか？  
「もっと簡潔に」「関西弁で」程度で止まっていませんか？

私はエンジニアとして、AIを単なるチャットボットではなく、自分の思考を拡張する\*\*「外付けの脳（Exocortex）」として扱いたいと考えています。そのために必要なのは、友達のような「人格」ではなく、私の認知特性（Cognitive Architecture）に完全にフィットする「通信プロトコル」\*\*です。

そこで、MBTI（思考の骨格）と12星座（出力のトーン＆マナー）を掛け合わせ、さらに使用するLLMモデル（Claude/Gemini/GPT）に合わせて最適化されたシステムプロンプトを自動生成するツール、"The Tailor" を開発しました。

## "The Tailor" とは？

"The Tailor" (ザ・テイラー) は、あなたの認知特性と好みの美学を入力するだけで、LLM（ChatGPT, Claude, Gemini）の設定画面に貼り付けるための「最強のシステムプロンプト」を仕立て上げるツールです。

## なぜ作ったのか？（課題感）

- **プロンプトエンジニアリングが面倒**: 毎回「あなたはプロの編集者です…」と書くのは非効率。
- **AIの人格が安定しない**: 「論理的であれ」と指示しても、どこかドライになりすぎる。あるいは逆に、優しすぎて本質を突いてこない。
- **モデルごとの癖**: ClaudeはXMLタグを好むし、Geminiは文脈を好む。これを手動で書き分けるのは苦痛。

これらを解決するために、\*\*「設定ファイル生成機（Config Generator）」\*\*として設計しました。

## 技術的なアプローチ：人格の「分離実装」

このツールの核心は、AIの個性を「人格」としてひとまとめにするのではなく、\*\*「機能（Function）」と「装飾（Style）」\*\*に分離した点にあります。

### 1\. 骨格：MBTIによる「認知プロトコル」の定義

MBTI（16タイプ）を、「性格」ではなく\*\*「情報処理の優先度」\*\*として定義し直しました。

| タイプ | 特徴 |
| --- | --- |
| INTJ | 結論ファースト、構造化、冷徹な戦略（Strategy First） |
| ENFP | 発想の飛躍、メタファー多用、ブレインストーミング（Possibility First） |
| ISTP | 実装重視、最小限の理論、トラブルシュート（Execution First） |

これにより、AIは「INTJのフリ」をするのではなく、\*\*「INTJユーザーにとって最も摩擦の少ない通信方式」\*\*で出力するようになります。

### 2\. 皮膚：星座による「トーン＆マナー」の注入

論理だけでは味気ないため、12星座のアーキタイプを\*\*「語彙フィルター（Vocabulary Filter）」\*\*として実装しました。

- **牡羊座**: "先陣を切る" "着火する" といった、能動的でスピーディーな語彙。
- **蠍座**: "深淵" "本質" "毒と薬" といった、深掘りする語彙。

ここでのポイントは、AIに「私は蠍座です」と名乗らせるのではなく、あくまで\*\*「文体（Style Modifier）」として適用させている点\*\*です。これにより、「冷徹な論理（INTJ）」を語りながらも、「どこか詩的で重厚（蠍座）」という、ユニークな出力が生まれます。

### 3\. 気質：血液型による「ワークスタイル」の定義

MBTIが「思考回路」なら、血液型は\*\*「タスク処理のOS（Operating System）」\*\*のようなものとして定義しました。日本人に馴染み深いこの分類を、AIの「働き方」に反映させています。

| 血液型 | 特徴 |
| --- | --- |
| A型 | 細部へのこだわり、慎重な検証、丁寧な進行（Detail-Oriented） |
| B型 | 興味ドリブン、一点突破、マイペースな進行（Focus-Oriented） |
| O型 | 大局的な視点、実利重視、おおらかな進行（Pragmatic） |
| AB型 | 批評的視点、二面性、クールな分析（Critical） |

これにより、「同じINTJでも、A型ならより慎重に、B型ならより一点集中で」という、人間臭い挙動の揺らぎ（Variance）を実装しています。

### 4\. 制御：スライダーによる「動的チューニング」

性格診断だけでは「今の気分」に対応できません。そこで、以下の3つのパラメータをスライダー（0-100%）で調整可能にし、システムプロンプトに\*\*動的なルール（Dynamic Rules）\*\*を注入するようにしました。

**指摘の鋭さ (Critical Resilience)**

- **Low**: 心理的安全性を重視し、肯定的なサンドイッチ話法で接する（優しいメンター）。
- **High**: 一切の配慮を捨て、論理的欠陥のみを鋭く指摘する（冷徹なコンパイラ）。

**思考の具体度 (Granularity)**

- **Low**: 概念、哲学、抽象的な全体像を語る（アーキテクト視点）。
- **High**: 実装コード、具体的な手順書、数値のみを出力する（実装者視点）。

**発想の飛躍 (Chaos Factor)**

- **High**: 文脈を無視した水平思考（Lateral Thinking）や、偶発的なアイデア結合を強制する（カオスな壁打ち相手）。

この「性格（静的）」×「パラメータ（動的）」の掛け合わせこそが、既存のプロンプト生成ツールにはない、"The Tailor" 独自のマトリクスです。

### 5\. 最適化：モデル別プロンプト生成

選択したターゲットモデルに合わせて、出力されるプロンプトの\*\*構文（Syntax）\*\*を動的に書き換えています。

| モデル | 形式 |
| --- | --- |
| Claude 4.6 | Anthropicが推奨する XMLタグ (`<instruction>`, `<role>`) を用いて、厳密に構造化 |
| Gemini 3 | Googleモデルが得意とする Context & Narrative （文脈重視）形式 |
| GPT-5.2 | シンプルかつ明確な Markdownヘッダー 形式 |

これにより、同じ設定でも、各モデルのポテンシャルを最大限に引き出すプロンプトが出力されます。

## Tech Stack

| カテゴリ | 技術 |
| --- | --- |
| Framework | Next.js 15 (App Router) |
| Language | TypeScript |
| UI Library | Tailwind CSS + shadcn/ui |
| Animation | Framer Motion |
| Deployment | Vercel |

UIデザインは、エンジニアが使っていて心地よい「IDE（統合開発環境）」や「設計図（Blueprint）」を意識し、ダークモード × グリッド背景を採用しました。

## 使い方

1. **Select Model**: 使用するAIモデル（Claude, GPT, Gemini）を選択。
2. **Define Profile**: あなたのMBTIと星座を選択。
3. **Fine Tune**: 「批判への耐性」「思考の具体度」「発想の飛躍」をスライダーで微調整。
4. **Generate**: 生成されたテキストをコピー。
5. **Paste**: 各AIの「Custom Instructions」や「System Prompt」設定に貼り付ける。

これだけです。チャット欄に毎回貼り付ける必要はありません。一度設定すれば、あなたのAIは「あなた専用の執事」に生まれ変わります。
