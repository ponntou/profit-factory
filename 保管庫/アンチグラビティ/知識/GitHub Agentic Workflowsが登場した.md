# 要約：GitHub Agentic Workflowsが登場した

## 概要
GitHub Actions上でAIコーディングエージェント（Copilot CLI、Claude Codeなど）をネイティブに動かし、リポジトリの雑務を自動化する「GitHub Agentic Workflows」機能（テクニカルプレビュー）の全体像を解説したスクラップ。

## 主要なポイント
- **仕組みの革新**:
  - 複雑なYAMLではなく、Markdown内に自然言語で「AIにしてほしいこと」を記述するだけでワークフローが動作するContinuous AI（継続的AI）の仕組み。
- **活用可能なユースケース**:
  - Issueの自動トリアージ・分類。
  - コード変更に伴うドキュメント（READMEなど）の自動更新。
  - PRと連携したコードの簡素化やテストカバレッジの向上。
  - 失敗したCIのログ解析と修正案の提示。
  - 定期的なリポジトリ活動状況（ステータスレポート）の自動作成。
- **セキュリティと運用**:
  - デフォルトは読み取り専用で、PR作成などの書き込み操作は「safe outputs」として明示的な許可が必要。
  - AIが勝手にコードをマージすることはなく、人間のレビュー・承認が必須となる安全設計。
  - プロンプトインジェクション対策も組み込まれている。
