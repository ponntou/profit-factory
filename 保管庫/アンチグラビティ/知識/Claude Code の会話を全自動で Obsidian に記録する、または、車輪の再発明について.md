# 要約：Claude Code の会話を全自動で Obsidian に記録する、または、車輪の再発明について

## 概要
複数の端末間で作業文脈（コンテキスト）を共有するため、Claude Codeの会話内容を自動でObsidianに記録するカスタムCLIツール（Go言語製）を開発した事例とその所感について解説された記事。

## 主要なポイント
- **自動記録の仕組み**: 
  - Claude CodeのHook機能（UserPromptSubmit, Stop, PostToolUse, SessionEnd）を活用。
  - 会話内容や実行コマンドをMarkdown形式でiCloud Drive上のObsidian Vaultへ自動保存。
  - `ls`や`cat`などの探索的コマンドはスキップし、有用な情報だけをCallout形式で記録。
- **端末間のシームレスな体験**:
  - セッション再開時やディレクトリ変更時に、過去ノートへのリンクを含む新しいノートが生成される。他の端末から開くだけで前回の文脈をすぐに確認可能。
  - セッション終了時にAIによる要約が自動で追加される。
- **「車輪の再発明」のパラダイムシフト**:
  - ニッチな個人ワークフローに合わせたツールを作る際、既存のOSSを探すよりも「AIに要件を伝えてゼロから書かせる（車輪の再発明をする）」方が早くて確実な時代になった。
  - 一般的なOSSにプルリクエストを送るよりも、AIを使って自分用にフォーク・改造するコストが劇的に下がっている。
